PKCE (Proof Key for Code Exchange) to zabezpieczenie chroniące aplikacje SPA/mobilne przed przechwyceniem kodu autoryzacyjnego.

  1. Klient generuje losowy sekret code_verifier (min. 43 znaki).
  2. Klient tworzy skrót SHA-256 z tego weryfikatora i koduje go jako Base64URL, uzyskując code_challenge.
  3. Podczas przekierowania logowania klient wysyła wyzwanie (challenge) do serwera autoryzacji. Serwer zapamiętuje wyzwanie.
  4. Po zalogowaniu klient otrzymuje kod autoryzacyjny i odsyła go na endpoint /oauth/token wraz z oryginalnym weryfikatorem code_verifier.
  5. Serwer haszuje otrzymany code_verifier i porównuje go z zapisanym wyzwaniem. Token jest wydawany tylko przy zgodności.

Dzięki temu potencjalny włamywacz, który przechwyciłby kod w punkcie 3, nie może go wymienić na token, ponieważ nie zna sekretu code_verifier.
